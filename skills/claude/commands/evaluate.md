# /alpha-evaluate

按单因子评价标准评估因子。所有收益类指标基于分位数突破策略（多分位数加权）。

## 策略逻辑

信号突破前/后 x% 分位数时开仓做多/做空，持有到信号回落至 50% 分位数时平仓。
分位数级别：0.1%, 0.5%, 1%, 5%, 10%

**分位数权重**：10% 权重最高（交易量大、最可靠），0.1% 最低。
- 10% 有效但 1% 无效 → 仍有价值
- 1% 有效但 10% 无效 → 可疑

| 分位数 | 权重 | 原因 |
|--------|------|------|
| 10% | 3.0 | 交易最多，包含所有更严格分位数的交易 |
| 5% | 2.0 | |
| 1% | 1.0 | |
| 0.5% | 0.5 | |
| 0.1% | 0.3 | 交易极少，统计意义最弱 |

## Input

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| specs | list[AlphaSpec] | 是 | 来自 /alpha-formalize 的输出 |
| backtest_results | dict | 否 | adapter 回测返回的原始指标 |

或通过 CLI：`astack evaluate --input specs.json --output reports.json`

## 评分维度（标准 1-8，每项 0~1 分）

**1. 预测能力（权重 3.0 — 最高）**
多分位数加权评估：
- IC 水平及稳定性（IC_mean, ICIR）
- 各分位数策略的加权夏普（10% 权重最高）
- 各分位数策略的加权正收益率
- 各分位数策略的加权胜率
- 收益单调性：10% 总收益 ≥ 5% ≥ 1%（因为宽松分位数包含更多交易）

**2. 多持仓周期稳健性（权重 1.0）**
在一组相邻持有期上，各 horizon 的分位数策略夏普是否为正。

**3. 年度一致性（权重 1.0）**
多分位数加权的逐年收益：每年对所有分位数加权投票"该年是否正收益"。不能依赖少数年份。

**4. 经济含义与交易逻辑（权重 1.5）**
能解释"为什么偏差存在"，非纯拟合。由 LLM 或人工评分。

**5. 参数简洁性（权重 0.8）**
核心参数 ≤ 3 个。

**6. 跨时间/跨品种稳健性（权重 1.0）**
多分位数加权的分品种收益：每个品种对所有分位数加权投票"该品种是否正收益"。

**7. 创新性（权重 0.8）**
非旧因子机械变形。由 LLM 或人工评分。

**8. 表达简洁可审查（权重 0.7）**
干净、可复现、可审查。

## 理想因子模板（标准 9）

清晰经济含义，无过拟合，各分位数策略收益为正且单调，每年正收益，高夏普，IC 高且稳定，分品种均衡。

## 否决条件（标准 10）

| 编号 | 条件 | 处理 |
|------|------|------|
| 10.1 | 存在未来数据（IC > 0.15） | **一票否决** → fail |
| 10.2 | 多分位数加权方向严重偏向单方向 | penalty -0.1 |
| 10.3 | IC 接近零 + 分组混乱 | penalty -0.1 |
| 10.4 | 参数 > 5 个 | penalty -0.1 |
| 10.5 | 近两年表现差或回撤 > 30% | penalty -0.1 |

## 验证协议（标准 11）

7:2:1 train/val/test → val 效果接近 train（sharpe 比值 ≥ 0.7）→ test 确认 → 入库。

## Failure Cases
- 无回测数据 → 仅做定性评估，其余标注"待回测补充"
- 触发 10.1 → 直接 fail
- 无分位数数据 → fallback 到聚合 sharpe，标注"需分位数策略数据"
